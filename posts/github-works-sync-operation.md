# GitHub Works SyncでWorks下書きを作る運用

## この記事で伝えたいこと

GitHub Works Syncを使うと、GitHubリポジトリをもとにWorksの下書きを作れます。

最初から完成したポートフォリオを書くのではなく、リポジトリ情報を入口にして、あとから内容を育てる運用にできます。

## 基本の流れ

```text
GitHub usernameを設定
↓
public repositoryを同期
↓
Works化したいrepoを選ぶ
↓
Portfolio下書きを作成
↓
課題・工夫・学びを追記
↓
公開
```

## 自動入力できる情報

GitHubから取得できる情報だけでも、下書きの土台になります。

- リポジトリ名
- description
- GitHub URL
- homepage
- main language
- topics
- pushed_at

## 手で追記すべき情報

一方で、GitHubだけでは分からない情報もあります。

- なぜ作ったのか
- どんな課題があったのか
- どこを工夫したのか
- 何を学んだのか
- 今後どう改善するのか

ここがWorksとしての価値になります。

## 運用のポイント

Worksは完成後に一度だけ書くものではなく、プロジェクトの成長に合わせて更新するページとして扱うと良いです。

## まとめ

GitHub Works Syncは、Works作成の入口を軽くするための機能です。

リポジトリ情報で下書きを作り、人間が背景や学びを足すことで、実績として伝わるWorksになります。
